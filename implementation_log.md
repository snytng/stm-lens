# Stm Lens for astah* 実装ログ

## 現状のステータス
*   **完了**:
    1.  プロジェクトのセットアップ（Maven, SDK）。
    2.  仕様・設計の策定 (`requirements.md`, `specification.md`, `design.md` 更新)。
    3.  UIスケルトンの実装 (`StmAnalysisView` 作成)。
    4.  **ロジックの実装**: `SimulationEngine` クラスに、初期状態探索、遷移取得、状態更新のロジックを実装。
    5.  **UIとロジックの結合**: `StmAnalysisView` から `SimulationEngine` を呼び出し、画面操作でシミュレーションが動くようにする。
    6.  **図のハイライト機能の実装**: `DiagramHighlighter` クラスを作成し、現在の状態（黄色背景・緑枠）および直前の遷移（マゼンタ）を図上で強調表示する。
    7.  **アクション(Entry/Do/Exit/Transition)の実装**: 状態遷移時に、遷移元/遷移先状態の各種アクションおよび遷移自体のエフェクトをログに出力する。
    8.  **アクションログ表示切替機能の実装**: Entry/Do/Exit/Transitionアクションログの表示・非表示を切り替えるチェックボックスを追加する。
    9.  **ガード条件の実装**: ガード条件を持つ遷移を、条件をラベルに含んだ別のボタンとして表示する。
    10. **リファクタリング**: `StmAnalysisView` 内のロジック（履歴保持など）の整理や、コードの可読性向上を行う。
    11. **複合状態の実装**: 状態の入れ子構造に対応し、Entry/Exitアクションや内部の初期状態への遷移を正しく処理する。
    12. **平行状態の実装**: 複数のリージョンが同時にアクティブになる状態を正しく処理する。また、複合状態・平行状態の親枠線の強調表示や、初期状態通過時の強調表示を追加。
    13. **複合状態退出時のハイライト強化**: 複合状態から遷移する際、親状態と内部の直前状態の両方をマゼンタでハイライトする。
    14. **イベントと遷移のカラーペアリング**: 遷移可能なイベントボタンと図上の遷移線を同じ色で表示し、視認性を向上させる。ボタンの文字色も背景色に合わせて調整。
    15. **履歴状態の実装**: 浅い履歴(H)と深い履歴(H*)に対応し、状態を復元できるようにする。
    16. **デバッグ出力機能の実装**: 現在の状態、履歴、モデル構造、ログをクリップボードにコピーする機能を実装し、トラブルシューティングを容易にする。
    17. **UML仕様適合性調査**: 現状の実装がUML仕様（遷移順序、複合状態、履歴など）に概ね適合していることを確認。
    18. **条件分岐（ジャンクション/選択）の実装**: ガード条件の評価タイミングや分岐ロジック（静的/動的）をUML仕様に基づき実装する。
    19. **遷移パスのLCA計算ロジック修正**: 遷移パスが外部の擬似状態を経由する場合でも、パス全体を考慮してLCA（最下位の共通の親）を正しく計算し、適切なEntry/Exitアクションが実行されるようにする。
    20. **テスト記録・再生機能の実装**: イベントシーケンスを記録し、タグ付き値に保存・再生する機能を実装する。
    21. **時間遷移イベント（タイマー）の実装**: `tm(msec)` 形式のイベントに対応する。実時間モードと高速モード（待ち時間スキップ）の切り替え機能を実装し、効率的な動作確認を可能にする。
    22. **タイマーモードの動的切替対応**: シミュレーション実行中にNormalモードとFastモードを切り替えた場合、現在待機中のタイマーに対して即座にモード変更を適用する（例：Normal→Fastへの切替時に、残りの待ち時間をスキップして即時発火させる）。
    23. **並行状態の同時遷移（Run-to-Completion）の実装**: 1つのイベントで複数の並行リージョンが同時に遷移する挙動に対応する（現在は1つずつしか動かない制限がある）。**※タイマーイベントの残り時間計算（Entry Timeの記録）も合わせて実装する。**
        *   **実装メモ (astah* API仕様)**:
            *   並行状態の領域 (Region) は `getSubvertexes()` では取得できない場合があり、`getStateRegions()` を併用する必要がある。
            *   APIスクリプトでは `Region` を直接作成できないため、テストモデルでは `State` で代用している。実装コード (`drillDown` 等) は両方のパターンに対応させた。
            *   モデル構造のダンプ時、`getContainer()` のオブジェクト同一性判定 (`equals`) が厳密すぎて失敗する場合があるため、ID比較や名前比較を併用して階層構造を正しく判定するようにした。
    24. **異常状態の検出とエラー表示**: 状態遷移実行中に、モデルの整合性が取れない状態（例: 非並行状態でカレントステートが複数存在するなど）を検知した場合、シミュレーションを停止し、エラーメッセージを表示する。
    25. **単体テスト環境の構築と初期テストの実装**: JUnit 5を導入し、`SimulationEngine` の基本的なロジック（初期状態探索など）を検証する単体テストを作成する。
    26. **実行履歴のタイムトラベルデバッグ機能の実装**: 実行したイベントシーケンスの履歴（スナップショット）を保持し、ナビゲーションボタンで前後の状態に移動・復元できる機能を追加。過去に戻った状態からの別ルート実行（履歴分岐）にも対応。
    27. **UIのコンパクト化（レイアウト整理）**: TitledBorderの廃止、コントロールの1行集約、テスト機能の折りたたみ、イベントとログの左右分割（`JSplitPane`）を行い、モデルの表示領域を広げ、人間工学的な見やすさを向上させた。
    28. **シミュレーションエンジンの安定性向上 (Bug Fixes)**: 並行状態からの退出時のバグ、履歴状態の復元時のバグ、および astah* API の暗黙の領域構造に起因する二重進入（亡霊状態）バグを根本的に修正し、複雑な状態遷移でもカレントステートを見失わずに正しく実行できるように改善した。
    29. **内部遷移（Internal Transition）の実装**: 状態から退出・進入せずにアクションのみを実行する内部遷移に対応した。astah* APIで通常の遷移（自己遷移）と見分けがつかない問題や、イベントが二重に取得されてしまう罠を回避し、正確なシミュレーションを実現した。
    30. **テキストベースの自動テスト実行とアサーション機能の実装**: テキスト形式のテストスクリプト（`Target:`, `Fire:`, `AssertState:`, `AssertLog:`）をUIに貼り付けて一括実行し、合否判定を行う機能を追加した。複数図の連続テスト（リグレッションテスト）にも対応。
    31. **自動テストUIのレイアウト改善**: テキストベースの自動テスト機能のUIを改善。スクリプト入力と結果表示を左右分割にし、IDE風の実行ボタンとステータス表示を導入することで、よりコンパクトで使いやすいレイアウトを実現した。合わせて、テスト実行時の堅牢性向上や回帰テストスクリプトの整備も行った。
    32. **テスト記録(T20)と自動テスト(T30)の統合**: UIでのシミュレーション記録（Record）や履歴からのスクリプト一括生成（Generate）機能を実装し、手動操作をT30のテストスクリプト形式で自動生成・保存するように改修。テスト管理エンジンをスクリプトベースに一本化した。
