# Stm Lens for astah* 実装ログ

## 現状のステータス
*   **完了**:
    1.  プロジェクトのセットアップ（Maven, SDK）。
    2.  仕様・設計の策定 (`requirements.md`, `specification.md`, `design.md` 更新)。
    3.  UIスケルトンの実装 (`StmAnalysisView` 作成)。
    4.  **ロジックの実装**: `SimulationEngine` クラスに、初期状態探索、遷移取得、状態更新のロジックを実装。
    5.  **UIとロジックの結合**: `StmAnalysisView` から `SimulationEngine` を呼び出し、画面操作でシミュレーションが動くようにする。
    6.  **図のハイライト機能の実装**: `DiagramHighlighter` クラスを作成し、現在の状態（黄色背景・緑枠）および直前の遷移（マゼンタ）を図上で強調表示する。
    7.  **アクション(Entry/Do/Exit/Transition)の実装**: 状態遷移時に、遷移元/遷移先状態の各種アクションおよび遷移自体のエフェクトをログに出力する。
    8.  **アクションログ表示切替機能の実装**: Entry/Do/Exit/Transitionアクションログの表示・非表示を切り替えるチェックボックスを追加する。
    9.  **ガード条件の実装**: ガード条件を持つ遷移を、条件をラベルに含んだ別のボタンとして表示する。
    10. **リファクタリング**: `StmAnalysisView` 内のロジック（履歴保持など）の整理や、コードの可読性向上を行う。
    11. **複合状態の実装**: 状態の入れ子構造に対応し、Entry/Exitアクションや内部の初期状態への遷移を正しく処理する。
    12. **平行状態の実装**: 複数のリージョンが同時にアクティブになる状態を正しく処理する。また、複合状態・平行状態の親枠線の強調表示や、初期状態通過時の強調表示を追加。
    13. **複合状態退出時のハイライト強化**: 複合状態から遷移する際、親状態と内部の直前状態の両方をマゼンタでハイライトする。
    14. **イベントと遷移のカラーペアリング**: 遷移可能なイベントボタンと図上の遷移線を同じ色で表示し、視認性を向上させる。ボタンの文字色も背景色に合わせて調整。
    15. **履歴状態の実装**: 浅い履歴(H)と深い履歴(H*)に対応し、状態を復元できるようにする。
    16. **デバッグ出力機能の実装**: 現在の状態、履歴、モデル構造、ログをクリップボードにコピーする機能を実装し、トラブルシューティングを容易にする。
    17. **UML仕様適合性調査**: 現状の実装がUML仕様（遷移順序、複合状態、履歴など）に概ね適合していることを確認。
## 次のステップ
    18. **条件分岐（ジャンクション/選択）の実装**: ガード条件の評価タイミングや分岐ロジック（静的/動的）をUML仕様に基づき実装する。
        *   **次回作業メモ**: テストケース(No.17)とモデル作成スクリプト(`test/scripts/create_test_model_no17.js`)は準備完了。
        *   **実装方針**: `getAvailableTransitions` を拡張し、遷移先がジャンクション等の擬似状態である場合、再帰的にその先の遷移を探索してパスを収集するロジックを実装する。
    19. **時間遷移イベント（タイマー）の実装**: `tm(msec)` 形式のイベントに対応する。実時間モードと高速モード（待ち時間スキップ）の切り替え機能を実装し、効率的な動作確認を可能にする。
    20. **テスト記録・再生機能の実装**: イベントシーケンスを記録し、タグ付き値に保存・再生する機能を実装する。
    21. **並行状態の高度な実装（Run-to-Completion）**: 1つのイベントで複数のリージョンが同時に遷移する挙動や、フォーク/ジョイン擬似状態、内部遷移に対応する。（UML仕様適合性調査での課題）
