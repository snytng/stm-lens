# Stm Lens for astah* 仕様書

## 1. コンセプト
astah*で作成した状態マシン図を、その場で動作（シミュレーション）させることで、設計の妥当性を早期に検証する。
静的な図だけでは見落としがちな「振る舞い」の欠陥（遷移の抜け漏れ、デッドロック、意図しない遷移など）を発見し、手戻りを削減する。

## 2. 入力仕様（astah*上の記述ルール）
本プラグインは、astah*上で選択されている「状態マシン図」を解析対象とする。

*   **対応要素**:
    *   **状態 (State)**: 通常の状態。
    *   **擬似状態 (Pseudostate)**:
        *   **開始状態 (Initial)**: シミュレーション開始点として必須。
        *   **終了状態 (Final)**: シミュレーション終了点。
    *   **遷移 (Transition)**: 状態間の移動。
        *   **トリガー (Event)**: 遷移を引き起こすイベント名。
        *   **ガード (Guard)**: 遷移条件（今回は簡易的にTrue/Falseを選択または無視する仕様とする）。
        *   **アクション (Action)**: 遷移時に実行される処理（ログに出力）。

## 3. 主要機能
### 3.1. シミュレーション制御
*   **開始 (Start)**: 選択中の状態マシン図の「開始状態」からシミュレーションを開始する。
*   **リセット (Reset)**: シミュレーションを初期状態に戻す。

### 3.2. イベント駆動遷移
*   現在の状態（カレントステート）から遷移可能なイベント（トリガー）を自動的に抽出し、ユーザーに提示する。
*   ユーザーがイベントを選択することで、状態遷移を実行する。

### 3.3. 状態の可視化
*   **カレントステート表示**: 現在の状態をUI上にテキストで表示するほか、astah*の図上で該当する状態をハイライト（色変更）する。
*   **実行ログ**: 遷移の履歴（From -> Event -> To）やアクションの実行をログとして表示する。

## 4. UI仕様
astah*の拡張タブ (Extra Tab) に以下の構成で表示する。

*   **ヘッダーエリア**:
    *   [Start] ボタン: シミュレーション開始。
    *   [Reset] ボタン: 初期化。
    *   状態表示ラベル: "Current State: [状態名]"
*   **イベントエリア**:
    *   現在の状態で発火可能なイベントをボタンとして一覧表示する。
    *   遷移不可能な場合は「No Events」等を表示。
*   **ログエリア**:
    *   テキストエリア形式でシミュレーションの履歴を表示する。
