# 貢献ガイドライン (Contributing Guidelines)

Stm Lens for astah* プロジェクトへの貢献に興味を持っていただきありがとうございます！
バグ報告、機能追加の提案、ドキュメントの改善、コードの修正など、あらゆる形での貢献を歓迎します。

## 1. バグ報告 (Bug Reports)
バグを発見した場合は、GitHubの Issues に報告してください。
報告の際は、以下の情報を含めていただけると助かります。

*   **再現手順**: バグを再現するための具体的なステップ。
*   **期待される動作**: 本来どうあるべきだったか。
*   **実際の動作**: 実際に何が起きたか（エラーメッセージやスクリーンショットなど）。
*   **環境情報**: astah*のバージョン、OS、Javaのバージョンなど。
*   **サンプルモデル**: 可能であれば、問題を再現できる最小限の `.asta` ファイルやスクリーンショット。

## 2. 機能要望 (Feature Requests)
新しい機能のアイデアや改善案がある場合も、GitHubの Issues に投稿してください。
「どのような問題を解決したいか」「どのような価値があるか」を明確に記述してください。

## 3. プルリクエスト (Pull Requests)
コードの修正や機能追加を行う場合は、以下の手順でプルリクエスト (PR) を送ってください。

1.  リポジトリを **Fork** します。
2.  作業用のブランチを作成します (`git checkout -b feature/your-feature-name` や `fix/bug-fix-name`)。
3.  変更を行い、コミットします。コミットメッセージは変更内容がわかるように記述してください。
4.  あなたのForkリポジトリにプッシュします (`git push origin feature/your-feature-name`)。
5.  本リポジトリに対して **Pull Request** を作成します。

### 開発のヒント
*   **コーディング規約**: 既存のコードスタイルに従ってください。
*   **テスト**: 新しい機能を追加した場合は、動作確認用のテストモデル作成スクリプト (`test/scripts/*.js`) とテスト手順 (`test/test_cases.md`) を追加・更新してください。
*   **ドキュメント**: 必要に応じて `README.md` や `usage.md` などのドキュメントも更新してください。

## 4. 開発環境のセットアップ
本プラグインの開発には以下の環境が必要です。

*   **JDK**: Java 8 またはそれ以降 (astah*のバージョンに合わせることを推奨)
*   **Maven**: ビルドツール
*   **astah* Professional / UML**: 動作確認用
*   **astah* Plug-in SDK**: Mavenリポジトリとして設定が必要な場合があります。
