# Stm Lens for astah* ユーザーガイド

## 1. 概要
Stm Lens for astah* は、astah* Professional / UML 上で作成した状態マシン図をシミュレーションし、その振る舞いを検証するためのプラグインです。

## 2. インストール方法
1.  リリースページから最新のプラグインファイル (`stm-lens-x.x.x.jar`) をダウンロードします。
2.  astah* を起動し、プラグインファイルをウィンドウにドラッグ＆ドロップします。
3.  astah* を再起動します。
4.  [表示] メニュー -> [拡張ビュー] -> [Stm Analysis] を選択してビューを表示します。

## 3. 基本的な使い方

### 3.1. 準備
1.  astah* を起動し、状態マシン図を含むプロジェクトを開きます。
2.  構造ツリーまたは図上で、シミュレーション対象としたい **状態マシン図** を開きます（アクティブにします）。
3.  astah* の拡張ビューエリアにある **[Stm Analysis]** タブを選択します。

### 3.2. シミュレーションの開始
1.  [Stm Analysis] タブ内の **[Start]** ボタンをクリックします。
2.  図上の **初期状態 (Initial Pseudostate)** からシミュレーションが開始されます。
3.  現在の状態（カレントステート）が図上で **黄色背景・緑枠** でハイライトされます。
4.  画面下部のログエリアに `Simulation started.` と表示されます。

### 3.3. 状態遷移の実行
1.  現在の状態から遷移可能なイベント（トリガー）が、画面中央の **Events** エリアにボタンとして表示されます。
    *   ガード条件がある場合、ボタンのラベルに `イベント名 [条件]` の形式で表示されます。
    *   遷移可能なイベントボタンと同じ色の線で、図上の遷移矢印もハイライトされます。
2.  実行したいイベントのボタンをクリックします。
3.  状態遷移が実行され、新しいカレントステートに移動します。
    *   遷移元・遷移先の Entry/Do/Exit アクションや、遷移自体のエフェクトがログに出力されます。
    *   図上のハイライトが更新されます（直前の遷移経路は **マゼンタ色** で表示されます）。

### 3.4. シミュレーションのリセット
1.  **[Reset]** ボタンをクリックすると、シミュレーションが終了し、初期状態に戻る準備が整います。
2.  図上のハイライトがすべてクリアされます。

### 3.5. タイムトラベルデバッグ
1. シミュレーション実行中に、イベントエリア上部のナビゲーションボタンを使用して履歴を行き来できます。
    *   `[|<<]` : 最初の状態へジャンプします。
    *   `[<]` : 1ステップ前の状態へ戻ります。
    *   `[>]` : 1ステップ次の状態へ進みます。
    *   `[>>|]` : 最後の状態へジャンプします。
2. 過去の状態に戻ると、カレントステートや図のハイライトが当時の状態に復元されます。
3. 過去に戻った状態で新しいイベントを実行すると、それ以降の未来の履歴は破棄され、新しい実行履歴として分岐（上書き）されます。

## 4. 高度な機能

### 4.1. アクションログの表示切替
*   画面上部コントロールパネルの **[Show Actions]** チェックボックスをオフにすると、Entry/Do/Exit などの詳細なアクションログを非表示にし、遷移イベントのみをシンプルに表示できます。

### 4.2. デバッグ情報の取得
*   画面上部の **[Copy Debug]** ボタンをクリックすると、現在のシミュレーションエンジンの内部状態（カレントステート、履歴情報、モデル構造など）と実行ログをクリップボードにコピーできます。
*   開発者への報告や、複雑な挙動の解析に役立ちます。

### 4.3. 複合状態・履歴状態の動作
*   **複合状態**: 内部に初期状態がある場合、自動的に内部へ遷移します。
*   **履歴状態 (H, H*)**: 複合状態から抜けた後、履歴状態への遷移で戻ると、以前の状態（浅い履歴または深い履歴）が復元されます。

### 4.4. 条件分岐 (Junction / Choice)
*   ジャンクションや選択（Choice）擬似状態を経由する遷移は、最終的な到達先ごとにボタンが分かれて表示されます。
*   例: `e1 [x > 0]` と `e1 [else]` のように、ガード条件ごとのパスを選択して実行できます。

### 4.5. 内部遷移 (Internal Transition)
*   状態に「内部遷移」が定義されている場合、そのイベントボタンをクリックしても状態からの退出・進入（カレントステートの変更）は発生しません。
*   対象状態の Entry / Exit アクションは実行されず、内部遷移自身に定義されたアクションのみがログに出力されます。
*   子状態に滞在している場合でも、親状態に定義された内部遷移イベントを実行することが可能です。

## 5. テストスクリプトの作成・実行機能 (自動テスト)
※ テスト機能を使用するには、画面上部の **[▼ Test Tools]** ボタンをクリックしてテストパネルを展開してください。

### 5.1. テストの作成 (Record / Generate)
テストスクリプトは、エディタに直接手書きするほかに、以下の2つの方法でUI操作から自動生成できます。
*   **リアルタイム記録 (Record)**:
    1.  **[Record]** ボタン（トグル）をオンにします。
    2.  イベントボタンをクリックしてシミュレーションを進めると、画面右側のエディタに `Fire: イベント名` と `AssertState:` がリアルタイムで追記されていきます。
    3.  ※ 過去の状態に戻る（タイムトラベル）と、安全のため記録は自動停止します。
*   **履歴から一括生成 (Generate)**:
    1.  自由にシミュレーションを操作し、正しい遷移パスを探します（戻ったり進んだりしても構いません）。
    2.  目的の状態に到達したら **[Generate]** ボタンをクリックします。
    3.  初期状態から現在地までの確定した実行履歴から、完全なテストスクリプトが一括で生成・出力されます。

### 5.2. テストの保存とロード
1.  エディタにスクリプトがある状態で **[Save]** ボタンをクリックし、名前をつけて保存します（モデルのタグ付き値として保存されます）。
2.  **Saved Tests:** のドロップダウンから保存済みのテストを選択すると、そのスクリプトが即座にエディタにロードされます。

### 5.3. テストの実行と管理
1.  エディタ上部の **[▶ Run Script]** ボタンをクリックすると、表示されているスクリプトが一括実行されます。
2.  実行結果（Pass/Fail）と詳細なアサーション結果が右側の結果エリアに表示されます。
3.  **[Rename]** / **[Delete]**: ドロップダウンで選択中のテストケースの名前変更・削除を行います。

## 6. 時間遷移イベント（タイマー）

### 6.1. 時間イベントの記述
*   遷移のトリガー（イベント名）に `tm(1000)` のように記述すると、1000ミリ秒（1秒）経過後に発火するタイマーイベントとして扱われます。
*   状態に進入後、指定時間が経過すると自動的に遷移します。
*   タイムアウト前に別のイベントで遷移した場合、タイマーはキャンセルされます。

### 6.2. 高速モード (Fast Mode)
*   画面上部の **[Fast Mode]** チェックボックスをオンにすると、タイマーの待ち時間を無視して即座に遷移を実行します。
*   長い待ち時間を含むモデルの動作確認を効率化できます。
*   **動的切替**: タイマー待機中にチェックボックスをオンにすると、残りの待ち時間をスキップして即座にイベントが発火します。逆にオフにすると、その時点から実時間での待機に戻ります（ただし、既に発火済みのイベントは戻りません）。

## 7. トラブルシューティング
*   **【異常検出】エラーダイアログが表示された場合**:
    *   シミュレーションの実行中に、状態マシンの内部状態に矛盾（例: 非並行状態なのに複数のカレントステートが存在するなど）を検知した場合、安全のためにシミュレーションを強制停止し、エラーダイアログを表示します。
    *   このエラーはモデルの書き方ではなく、プラグイン自体の制限やバグに起因する可能性があります。[Copy Debug] ボタンで情報をコピーし、開発者にご報告ください。

## 8. 注意事項
*   シミュレーションを実行するには、図上に必ず **初期状態 (Initial Pseudostate)** が1つ存在する必要があります。
