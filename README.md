# Stm Lens for astah*

## 概要
**Stm Lens for astah*** は、astah* Professional / UML 上で作成した状態マシン図（State Machine Diagram）をシミュレーションし、その振る舞いを検証するためのプラグインです。

静的な図だけでは見落としがちな「振る舞い」の欠陥（遷移の抜け漏れ、デッドロック、意図しない遷移など）を設計段階で発見し、手戻りを削減することを目的としています。

## 特徴

### 1. インタラクティブなシミュレーション
*   astah*上で作成した状態マシン図をそのまま実行できます。
*   現在の状態（カレントステート）から遷移可能なイベントがボタンとして表示され、クリックするだけで遷移を実行できます。
*   図上で現在の状態や遷移経路がハイライトされ、視覚的に動作を確認できます。

### 2. 高度なUML機能のサポート
*   **複合状態 (Composite State)**: 状態の入れ子構造に対応し、Entry/Exitアクションや内部遷移を正しく処理します。
*   **履歴状態 (History State)**: 浅い履歴(H)と深い履歴(H*)に対応し、状態の復元をシミュレーションできます。
*   **条件分岐 (Junction / Choice)**: ガード条件による分岐ロジックを検証できます。
*   **並行状態 (Orthogonal State)**: 複数のリージョンが同時にアクティブになる状態に対応しています（※現在は簡易実装）。

### 3. 詳細な実行ログ
*   状態遷移の履歴だけでなく、Entry/Do/Exitアクションの実行順序もログとして出力されます。
*   アクションログの表示/非表示を切り替え、遷移の流れだけを追うことも可能です。

### 4. デバッグ支援
*   現在の内部状態（カレントステート、履歴情報など）やモデル構造をクリップボードにコピーする機能を搭載しており、複雑な挙動の解析やバグ報告に役立ちます。

### 5. テスト記録・再生
*   実行したイベントシーケンスを記録し、テストケースとしてモデル内に保存できます。
*   保存したテストケースを再生することで、モデル変更後の回帰テスト（リグレッションテスト）を容易に行えます。

## 動作環境
*   **astah* Professional** または **astah* UML** (バージョン 8.0以上推奨)
*   Java Runtime Environment (astah*に同梱されているもので動作します)

## 使い方
詳細な使い方は [usage.md](usage.md) を参照してください。

1.  シミュレーションしたい状態マシン図を開きます。
2.  [Stm Analysis] ビューの **[Start]** ボタンをクリックします。
3.  表示されるイベントボタンをクリックして状態遷移を進めます。
