## 実装待ちタスク (To Do)
- [x] B17. **[🚨 CRITICAL BUG] テスト生成時のガード条件欠落バグの修正**: RecordおよびGenerate機能でテストスクリプトを自動生成する際、イベント名にガード条件（例: `[x > 0]` や `[else]`）が含まれず素のイベント名のみが出力されてしまう。これにより、ジャンクション等を含むモデルのテスト実行時（Run Script）に該当ボタンが見つからずエラーになる致命的な問題を修正する。
- [ ] B3. **フォーク（Fork）擬似状態の実装**: 1つの遷移を複数の並行リージョンへの遷移に分割するフォーク擬似状態に対応する。
- [ ] B4. **ジョイン（Join）擬似状態の実装**: 複数の並行リージョンからの遷移を同期し、全てが揃った時点で1つの遷移に統合するジョイン擬似状態に対応する。
- [ ] B7. **テストカバレッジ（N-Switch）の計測とテストケース自動生成**: シミュレーション実行時に遷移ペア（n-switch）のカバレッジを計測し、未達の遷移パスを自動的に実行するテストケースを生成・登録する機能を追加する。
- [ ] B9. **静的モデル検証（Lint）機能の実装**: シミュレーション実行前に、UMLの構文違反や論理的な問題（初期状態の欠落、到達不能な状態、非決定的な遷移など）を事前にチェックし、ユーザーに警告・エラーを提示する機能を追加する。
- [x] B10. **UIのコンパクト化（レイアウト整理）**: 画面の縦幅を節約し、図の表示領域を広げるためにボタンの配置を見直す。
- [x] B11. **テキストベースのテスト自動実行とアサーション機能**: テキスト形式のテストスクリプト（図の指定、イベント投入、期待される状態・ログのAssert）をUIに貼り付けて一括実行し、合否判定を行う機能を追加する。複数図の連続テスト（リグレッションテスト）にも対応する。
- [x] B12. **自動テストUIのレイアウト改善**: テキストベースの自動テスト機能について、スクリプト入力欄と結果表示欄が縦に並び画面を占有している問題を解決する。左右分割（JSplitPane）、IDE風の実行ボタンとステータス表示を採用し、よりコンパクトで使いやすいレイアウトに改善する。
- [x] B13. **テスト記録(T20)と自動テスト(T30)の統合**: UIでのシミュレーション記録（Record）を、T30のテキストテストスクリプト形式（`Target:`, `Fire:`, `AssertState:`）で自動生成・保存するように改修する。これにより、手動操作がそのままアサーション付きの回帰テストとして機能するようにし、テスト管理エンジンを一本化する。
- [x] B14. **Test Scriptクリアボタンの追加**: スクリプトエリアのテキストを頻繁に削除する手間を省くため、ワンクリックで内容を消去できるクリアボタン（ゴミ箱アイコン等）をUIに追加する。
- [x] B15. **テストロード時の上書き確認ダイアログの追加**: 保存済みのテストケースをドロップダウンから選択してロードする際、現在エディタに編集中のスクリプトが存在する場合は、誤上書きを防ぐ（および意図的な削除操作を省く）ための確認ダイアログを表示する。
- [ ] B16. **テスト記録仕様の見直し（常時Generate化）**: 途中からRecordを開始するユースケースの必要性を再考し、「シミュレーションのStartと同時に常に操作が記録（自動Generate）される」仕様への変更を検討する。手書きを前提とした難易度の高い仕様から脱却し、誰でも直感的にテストを作れる体験へ洗練させる。
